# ADR-002: Solana Integration Strategy

**Date:** 2026-07-03
**Status:** Approved

## Context

LucidDrop requires blockchain integration for:
- Real-money deposits and withdrawals
- Provably fair game verification
- Wallet authentication and management
- Transaction transparency and immutability

Multiple blockchain options were considered, with Solana chosen for its speed, low costs, and ecosystem support.

## Decision

We will integrate Solana blockchain using the following approach:

### Core Technology
- **@solana/web3.js v1.95.8**: Primary SDK for blockchain interaction
- **@solana/wallet-adapter-react & react-ui**: Frontend wallet connection
- **tweetnacl**: Cryptographic signature verification
- **bs58**: Base58 encoding for Solana addresses

### Key Components

1. **Solana Service (`solana.js`)**
   - Transaction processing (deposits, withdrawals)
   - Balance checking
   - Smart contract interaction
   - Network connection management

2. **Solana Listener (`solanaListener.js`)**
   - Real-time transaction monitoring
   - Deposit detection
   - Event processing

3. **Solana Stealth (`solanaStealth.js`)**
   - Private key management
   - Deposit address generation
   - Secure key storage

### Wallet Integration
- **Phantom Wallet**: Primary wallet supported
- **Wallet Adapter**: Multi-wallet support via Solana Wallet Adapter
- **Connection Flow**: Signature-based authentication

### Transaction Flow

**Deposit Flow:**
1. User requests deposit address
2. Backend generates stealth address
3. Listener monitors for incoming SOL
4. Balance credited to user account

**Withdrawal Flow:**
1. User initiates withdrawal
2. Backend validates request and balance
3. Transaction signed and broadcast
4. Balance updated after confirmation

### Smart Contract
- **LucidDrop.sol**: Solidity contract deployed on Solana
- **OpenZeppelin**: Used for secure contract patterns
- **Provably Fair**: On-chain seed verification

## Rationale

### Why Solana?
- **Speed**: 400ms block times for fast transactions
- **Low Fees**: Fractions of a cent per transaction
- **Ecosystem**: Strong gaming and DeFi ecosystem
- **Phantom Wallet**: Excellent user experience
- **Developer Tools**: Well-documented SDKs

### Alternatives Considered

| Blockchain | Speed | Cost | Ecosystem | Decision |

## Consequences

### Positive
- Fast, low-cost transactions for users
- Excellent wallet support (Phantom)
- Growing developer ecosystem
- Scalable for high-volume betting

### Negative
- Dependency on Solana network stability
- Learning curve for blockchain operations
- Private key management complexity
- Network congestion in extreme conditions

### Risks
- **Network Outages**: Solana has experienced outages; need fallback strategy
- **Security**: Private key management requires strict security practices
- **Regulatory**: Gambling regulations vary by jurisdiction

## Security Considerations

### Private Key Management
- Stored in encrypted `casino-wallet.json`
- Environment variables for sensitive keys
- Stealth wallet pattern for deposits

### Transaction Validation
- All transactions validated before processing
- Confirmation monitoring for finality
- Duplicate transaction prevention

### Authentication
- Wallet signature verification using tweetnacl
- JWT tokens for session management
- Rate limiting on wallet operations
