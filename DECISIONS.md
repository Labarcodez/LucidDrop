# LucidDrop Decisions

**Last Updated:** 2026-07-03

## Architecture Decisions

### Decision 1: Solana Blockchain Integration
**Date:** 2026-07-03
**Status:** Approved

**Context:** The platform requires a blockchain for real-money transactions, provably fair gaming, and wallet integration.

**Decision:** Use Solana blockchain with Web3.js for:
- Fast transaction processing (400ms block times)
- Low transaction fees (fractions of a cent)
- Strong ecosystem support
- Phantom wallet integration

**Alternatives Considered:**
- Ethereum: Higher fees, slower transactions
- Polygon: Good alternative but less established in gaming

### Decision 2: Technology Stack Selection
**Date:** 2026-07-03
**Status:** Approved

**Context:** Need a modern, scalable web application stack for real-time gaming.

**Decision:**
- **Frontend:** React 18 with Tailwind CSS and Zustand
- **Backend:** Node.js with Express
- **Database:** MongoDB Atlas
- **Real-time:** Socket.io
- **Smart Contracts:** Hardhat with OpenZeppelin

**Rationale:**
- React provides component reusability and performance
- Tailwind enables rapid UI development
- Zustand offers lightweight state management
- Node.js/Express for high-performance API
- MongoDB Atlas for managed database
- Socket.io for real-time updates
- Hardhat for robust contract development

---

### Decision 3: Provably Fair System
**Date:** 2026-07-03
**Status:** Approved

**Context:** Need to ensure game fairness and transparency for users.

**Decision:** Implement a provably fair system using seed generation and verification:
- Server seed generated and hashed
- Client seed provided by user
- Combined seeds determine game outcomes
- Results verifiable by users

**Rationale:** Builds trust, meets regulatory requirements, provides transparency.

---

### Decision 4: Real-time Architecture
**Date:** 2026-07-03
**Status:** Approved

**Context:** Games require real-time updates and user interaction.

**Decision:** Use Socket.io for WebSocket communication:
- Room-based event broadcasting
- Real-time game state updates
- Live chat functionality
- Leaderboard updates

**Rationale:** Socket.io provides reliable, event-driven communication with automatic reconnection.

---

### Decision 5: Wallet Authentication
**Date:** 2026-07-03
**Status:** Approved

**Context:** Users need to authenticate with their Phantom wallet.

**Decision:** Implement signature-based authentication:
- User signs a message with their wallet
- Backend verifies the signature
- JWT token issued for session

**Rationale:** Secure, user-friendly, leverages existing wallet infrastructure.
