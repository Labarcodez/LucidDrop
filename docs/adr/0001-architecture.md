# ADR-001: System Architecture

**Date:** 2026-07-03
**Status:** Approved

## Context

LucidDrop is a Solana-based casino platform requiring real-time gaming, secure blockchain transactions, and a responsive user interface. The system must handle high-frequency betting, real-time updates, and secure wallet integration.

## Decision

We have selected a layered architecture with clear separation of concerns:

### Architecture Layers

1. **Frontend Layer (React 18)**
   - Component-based UI architecture
   - Zustand for state management
   - Tailwind CSS for styling
   - Solana Wallet Adapter for wallet integration
   - Socket.io client for real-time updates

2. **Backend Layer (Node.js/Express)**
   - REST API for business operations
   - Socket.io server for real-time communication
   - JWT authentication with wallet signature verification
   - Service layer for business logic
   - Mongoose ODM for database operations

3. **Database Layer (MongoDB Atlas)**
   - Document-based storage for flexible schemas
   - Mongoose models for data access
   - Managed cloud database for production

4. **Blockchain Layer (Solana)**
   - Web3.js for blockchain interaction
   - Phantom wallet support
   - Stealth wallet for deposit management
   - Smart contract for on-chain logic

### Key Patterns

- **Service Layer Pattern**: Encapsulates business logic in services
- **Repository Pattern**: Mongoose models provide data access abstraction
- **Event-Driven Architecture**: Socket.io for real-time communication
- **Provider Pattern**: Context providers for wallet and authentication

## Rationale

- **Separation of Concerns**: Each layer has distinct responsibilities
- **Scalability**: Independent scaling of layers
- **Maintainability**: Clear boundaries between components
- **Modern Stack**: Current, well-supported technologies
- **Real-time Capability**: Socket.io enables live updates
- **Blockchain Integration**: Solana provides fast, low-cost transactions

## Consequences

### Positive
- Clear architecture boundaries
- Easy to add new features
- Independent testing of layers
- Good developer experience

### Negative
- Complexity of managing multiple layers
- Need for robust error handling between layers
- Higher initial setup overhead

## Alternatives Considered

### Alternative 1: Monolithic Architecture
- **Rejected**: Would limit scalability and maintainability

### Alternative 2: Microservices
- **Rejected**: Too complex for current scope, increased operational overhead

### Alternative 3: Single-Page Application with Serverless
- **Rejected**: WebSocket requirements make serverless challenging
